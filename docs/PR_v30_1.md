# v30.1 — Bascule v30 par défaut + parité v29 complétée

Cette PR fait basculer la v30 en interface par défaut, rattrape les derniers manques fonctionnels v29, et range les templates legacy dans `templates/legacy/` pour clarifier la structure. v29 reste 100 % accessible via opt-out client (`localStorage.prospup_ui_mode === 'v29'`) ou `?force_v29=1` sur l'URL.

## Résumé

- **APP_VERSION** : `30.0` → `30.1`
- **v30 est le mode par défaut** au login et sur les routes legacy (redirect client → v30 sauf opt-out)
- **22 templates legacy** déplacés dans `templates/legacy/` (via `git mv`)
- **Parité v29 → v30 estimée à ~95 %** (vs 58 % avant v30.1)
- Aucune route Flask supprimée, aucune migration DB, aucun endpoint `/api/*` modifié

## Changements groupés

### Sprint 2 — 7 manques fonctionnels P1

- **Sidebar v30** : sections Outils + Admin role-aware (Collab, Doublons, DC, Users, Snapshots, Journal, Métiers)
- **Prospects** : 4 KPI cards + colonnes Email / Push / Voir
- **Fiche candidat** : bloc Informations (10 champs)
- **Activity** : colonnes Entité + Détails
- **Stats** : 8 KPI + table Entreprises chaudes
- **Dashboard** : boutons `+ KPI manuel` + `Export` (redirect `/dashboard#kpi-manual`)

### Sprint 3 — 5 items P2 polish

- **Topbar v30** : menu avatar cliquable (Paramètres / Aide / Déconnexion)
- **/v30/prospects** : bannière relances en retard (dismissible)

- **/v30/focus** : bloc Tâches CRUD branché sur `/api/tasks`

### Partie 2 — Bascule v30 par défaut

- `app.py` : `/login` redirige vers `/v30/dashboard`
- `login.html` : redirect client post-login vers v30 sauf opt-out
- `static/js/v30/opt-in.js` : `autoRedirectToV30()` avec mapping 18 routes legacy → v30
- **Escape hatches** : `localStorage.prospup_ui_mode === 'v29'` (opt-out), `?force_v29=1` (bypass URL)

### Partie 2 — Move templates v29

- 22 fichiers déplacés dans `templates/legacy/` via `git mv`
- `app.py` : tous les `render_template("xxx.html")` → `render_template("legacy/xxx.html")`
- 20 templates : `{% extends "base.html" %}` → `{% extends "legacy/base.html" %}`
- `templates/v30/` et `templates/_partials/` intouchés
