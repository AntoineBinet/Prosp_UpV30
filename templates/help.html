1) Démarrage rapide

  1. Lancez app.py (double-clic sur start.bat ou commande python app.py).
  2. Ouvrez l'URL locale affichée (généralement http://127.0.0.1:8000).
  3. Ajoutez quelques entreprises puis des prospects (ou importez vos données JSON/Excel).
Données locales — Stockage SQLite sur votre PC. Aucune donnée envoyée sur Internet. Un snapshot automatique est créé chaque jour au premier lancement.
Tutoriel
  1. Connectez-vous (admin / admin par défaut).
  2. Ouvrez le Dashboard pour voir la vue d'ensemble.
Voir le Dashboard →

1b) Utilisation à distance (prospup.work)

Vous pouvez utiliser Prosp'Up en vous connectant à prospup.work depuis n'importe quel appareil (bureau, domicile, autre PC). Voici ce qu'il faut savoir :

  • Où tourne l'application ? Prosp'Up et vos données tournent sur un seul PC (celui sur lequel vous avez lancé app.py et éventuellement un tunnel Cloudflare). C'est ce PC qui « héberge » l'accès prospup.work.
  • Vos données (prospects, entreprises, candidats) sont stockées sur ce PC. En vous connectant à prospup.work, vous travaillez sur les mêmes données que si vous étiez devant ce PC.
  • Fonctions IA (Ollama) : pour que les boutons « Scrapping IA », « Ajout IA », « Générer avec Ollama », etc. fonctionnent, Ollama doit être installé et lancé sur le même PC qui héberge Prosp'Up. Si vous utilisez Prosp'Up depuis un autre ordinateur via prospup.work, l'IA s'exécute donc sur le PC serveur, pas sur votre machine. Si Ollama n'est pas démarré sur ce PC, les boutons IA afficheront une erreur ou une modale vide ; vous pouvez alors utiliser l'option « Copier » pour utiliser une autre IA (ex. ChatGPT) et coller le retour dans l'app.
  • En résumé : pour un utilisateur lambda qui se connecte à prospup.work à distance, tout fonctionne comme en local, à condition que sur le PC qui héberge Prosp'Up, Ollama soit lancé si vous voulez utiliser les fonctions IA.

  • KPI du jour : nombre de prospects, push envoyés, RDV planifiés, relances en retard, avec indicateurs de tendance (↑/↓) par rapport à la période précédente.
  • Graphique semaine : activité push et notes d'appel sur les derniers jours.
  • Relances en retard : liste directe des prospects dont le suivi est dépassé, avec accès rapide à la fiche.
  • Fil d'activité : flux chronologique de vos dernières actions (push, notes, changements de statut).
  • RDV à venir : prochains rendez-vous planifiés.
  • Pipeline : répartition visuelle de vos prospects par statut.

3) Prospects

La page principale de gestion de vos prospects (managers, responsables, décideurs chez vos clients potentiels).

  • Filtres avancés : recherche texte, entreprise, statut, pertinence (1–5 étoiles), relance (due / a une date / aucune), "avec téléphone", "avec email", "push envoyé".
  • Tri : cliquez sur les en-têtes de colonnes (Nom, Entreprise, Pertinence, Statut, Date de push…).
  • Sélection multiple : cochez plusieurs prospects pour appliquer un statut ou une pertinence en masse, exporter, ou supprimer.
  • Vue Tableau / Kanban : basculez entre une vue tableau classique et une vue Kanban qui organise les prospects par statut sous forme de colonnes glisser-déposer.
  • Vues sauvegardées : enregistrez vos combinaisons de filtres et tris favorites pour les retrouver en un clic. Créez autant de vues que nécessaire (ex: "Mes prospects chauds Lyon", "Relances de la semaine").
  • Vue Contacts : activez le mode contacts (?contacts=1 dans la sidebar) pour un affichage orienté annuaire avec téléphone et email en avant.
  • Export : export Excel (.xlsx) de la liste filtrée avec toutes les colonnes et données enrichies.
Tutoriel
  1. Ouvrez la page Prospects.
  2. Cliquez sur ⚙️ Filtres pour affiner par statut ou entreprise.
  3. Cliquez sur une ligne pour ouvrir la fiche détaillée (onglets Infos, Historique, Relance, etc.).
Ouvrir la liste Prospects →

4) Fiche prospect détaillée

Cliquez sur un prospect pour ouvrir sa fiche complète avec plusieurs onglets :

  • Infos : nom, fonction, entreprise, téléphone, email, LinkedIn, pertinence (étoiles), statut, tags techniques, notes libres.
  • Photo : uploadez une photo du prospect pour le reconnaître facilement.
  • Historique : timeline chronologique de toutes les interactions — push envoyés, notes d'appel, enrichissements IA.
  • Relance : définissez la prochaine date de relance (nextFollowUp). Le prospect apparaîtra dans la page Focus.
  • Catégorie Push : associez une catégorie push au prospect pour que le bon template Outlook s'ouvre automatiquement lors de l'envoi.
  • Meilleurs candidats : l'app compare automatiquement les tags du prospect avec les compétences de vos candidats et affiche les meilleurs matchs (score de correspondance en %).
  • Onglet RDV (si statut "Rendez-vous") : checklist de qualification RDV avec des thèmes de discussion pré-configurés, barre de progression, et cases à cocher pour préparer et suivre vos rendez-vous.
  • Actions rapides : bouton email (copie + ouverture template Outlook), vCard, scrapping IA, marquer comme fait.

5) Scrapping IA (enrichissement)

Enrichissez vos fiches prospect, entreprise et candidat grâce à l'IA locale Ollama (un seul clic) :

  1. Cliquez sur "🤖 Scrapping IA" : l'application envoie un prompt optimisé (données actuelles de la fiche) à Ollama tournant sur votre PC. La génération peut prendre quelques secondes.
  2. Une modale s'ouvre avec le résultat pré-rempli. Le système analyse le texte, détecte les champs (fonction, téléphone, email, LinkedIn, tags, notes…) et affiche une prévisualisation.
Prérequis — Ollama doit être installé et lancé sur le même PC que Prosp'Up. Modèle par défaut : llama3.2 (variables OLLAMA_MODEL, OLLAMA_URL). Si Ollama est indisponible, la modale s'ouvre vide : vous pouvez coller le retour d'une autre IA (ChatGPT, Claude).

Vous pouvez accepter ou ignorer chaque champ individuellement avant d'appliquer. L'enrichissement est tracé dans la timeline.

  • Prospect : enrichit la fonction, téléphone, email, LinkedIn, tags techniques, métier, pertinence, notes, et infos entreprise.
  • Entreprise : enrichit le secteur, effectif, téléphone standard, tags, notes d'activité, et peut détecter des managers clés à créer automatiquement comme nouveaux prospects.
  • Candidat : enrichit le rôle, compétences techniques, séniorité, localisation, LinkedIn, disponibilité, et TJM.
Tutoriel
  1. Ouvrez la liste Prospects et cliquez sur un prospect pour ouvrir sa fiche.
  2. Dans la fiche, cliquez sur le bouton 🤖 Scrapping IA (en haut ou dans les actions).
  3. Attendez la génération (ou collez un texte), puis analysez et acceptez les champs à appliquer.
Ouvrir Prospects (puis une fiche + Scrapping IA) →

5b) Ajout IA (Quick Add)

Le bouton « Ajout IA » (page Prospects ou barre d'actions) permet de créer un ou plusieurs prospects, entreprises ou candidats à partir de l'IA (Ollama) ou d'un retour collé manuellement. Flux en 3 étapes :

  1. Type + Contexte : choisissez ce que vous voulez ajouter (Prospect, Entreprise ou Candidat). Renseignez le champ Contexte (recommandé) : un lien LinkedIn, un nom et une entreprise, ou un court texte. Sans contexte, l'IA génèrera un exemple que vous pourrez modifier à l'étape suivante.
  2. Générer ou coller :
    • Générer avec Ollama (un seul / plusieurs) : envoie un prompt à Ollama sur le PC qui héberge Prosp'Up. Le résultat s'affiche dans la zone de texte. Vous pouvez modifier le JSON (nom, LinkedIn, téléphone, etc.) avant de cliquer Analyser.
    • Copier (un seul / plusieurs) : copie le prompt dans le presse-papier. Collez-le dans Ollama (ou une autre IA) sur votre machine, ajoutez votre lien ou contexte si besoin, puis collez le retour de l'IA dans la zone ci-dessous et cliquez Analyser.
  3. Aperçu et création : après analyse, les fiches détectées s'affichent. Décochez celles à ne pas créer, puis cliquez Créer.

À distance (prospup.work) : si vous êtes connecté via prospup.work depuis un autre PC, « Générer avec Ollama » utilise Ollama installé sur le PC qui héberge Prosp'Up. Si Ollama n'y est pas lancé, utilisez « Copier » et une IA locale ou en ligne, puis collez le retour dans l'app.

Tutoriel interactif

Le lien ci-dessous ouvre la page Prospects et affiche la modale « Ajout IA ». Choisissez un type, remplissez le contexte puis « Générer avec Ollama » ou « Copier ».

6) Entreprises

La liste de vos entreprises clientes / cibles, avec vue résumée :

  • Groupe + Site : organisez par groupe (ex: "Renault") et site/localisation (ex: "Vénissieux").
  • Tags techniques : stockez la stack technique de l'entreprise (C/C++, FPGA, AUTOSAR…). Ces tags sont utilisés par le moteur de matching sourcing.
  • Notes : contexte libre, interlocuteurs, pistes commerciales.
  • Résumé rapide : nombre de prospects, dernier push, RDV planifiés.
  • Fiche popup : ouvrez la fiche détaillée depuis la liste, sans quitter l'écran entreprises.
  • Scrapping IA : même workflow qu'en prospect (un clic avec Ollama).

7) Fiche entreprise détaillée

La fiche détaillée s'ouvre en popup depuis la page /entreprises et offre une vue complète de l'entreprise :

  • Informations : groupe, site, téléphone, tags, secteurs, notes — tout modifiable en ligne.
  • Prospects liés : liste de tous les prospects rattachés à cette entreprise, avec accès direct à chaque fiche.
  • Opportunités : gérez vos opportunités commerciales par entreprise. Chaque opportunité a un titre, un stade (Identification → Qualification → Proposition → Négociation → Gagné / Perdu), un candidat associé, un montant, et des notes. Idéal pour suivre vos placements en cours.
  • Timeline : historique chronologique de toutes les interactions — push, notes de prospects, événements, enrichissements IA.
  • Événements : ajoutez des événements manuels à la timeline (rencontres, appels, infos).

8) Focus & relances

La page Focus est votre to-do de prospection. Elle centralise tous les prospects ayant une date de relance (nextFollowUp) définie :

  • Segments : en retard (rouge), aujourd'hui (orange), demain, cette semaine, plus tard.
  • Actions rapides : décaler la relance, marquer "fait" (supprime la date de relance), ouvrir la fiche prospect.
  • Compteurs : nombre de relances par segment pour prioriser votre journée.
Conseil — À chaque interaction avec un prospect, définissez la prochaine relance. Votre Focus devient votre agenda de prospection.
Ouvrir la page Focus →

9) Calendrier

Vue calendrier mensuelle de vos RDV planifiés, relances et entretiens candidats :

  • Les RDV (prospects avec statut "Rendez-vous" et date RDV) sont affichés sur le calendrier avec l'icône 🤝.
  • Les relances planifiées (nextFollowUp) sont également visibles avec l'icône 🔄 (⚠️ si en retard).
  • Entretiens candidats EC1 : les entretiens de qualification (EC1) planifiés apparaissent avec l'icône 📞. La date provient de la checklist EC1 du candidat.
  • Entretiens candidats EC2 : les entretiens techniques (EC2) apparaissent avec l'icône 📞📞. La date est déterminée automatiquement depuis le passage du candidat au statut EC2.
  • Cliquez sur un événement pour accéder directement à la fiche prospect ou candidat (les EC1 ouvrent directement la section EC1 de la fiche candidat).
  • Navigation mois par mois ou vue semaine.
  • Modale détail du jour : cliquez sur un jour avec plusieurs événements pour voir la liste complète.
Astuce — Les EC1/EC2 sont automatiquement ajoutés au calendrier dès que vous planifiez un entretien dans la fiche candidat ou que vous changez le statut à EC2.
Ouvrir le Calendrier →

10) Sourcing candidats

Votre base candidats et outil de productivité pour le matching :

  • Pipeline candidats : liste complète avec filtres par statut (À sourcer, Contacté, Qualifié, En mission, Archivé), recherche, et tri.
  • Ajout rapide : créez un candidat avec nom, rôle, compétences, localisation, LinkedIn, source.
  • Import LinkedIn CSV : importez directement un export de contacts LinkedIn — le système mappe automatiquement les colonnes (prénom, nom, poste, URL…).
  • Ajouter via VSA : ajoutez un candidat à partir d’une fiche VSA (Suivi des candidats) en collant le contenu de la page puis en utilisant l’extraction Ollama. Voir ci-dessous.
  • Export CSV : exportez votre base candidats.

Ajouter un candidat depuis VSA

Si vous utilisez VSA (vsactivity) pour le suivi des candidats, vous pouvez importer une fiche dans Prosp’Up sans saisie manuelle :

  1. Sur VSA : ouvrez la fiche du candidat, puis sélectionnez tout le texte de la page (Ctrl+A ou Cmd+A) et copiez (Ctrl+C ou Cmd+C).
  2. Dans Prosp’Up : cliquez sur « 🔗 Ajouter via VSA » sur la page Sourcing. Collez éventuellement le lien de la fiche VSA (optionnel), puis collez le texte copié dans la zone prévue.
  3. Extraire : cliquez sur « Extraire avec Ollama ». Les champs (nom, rôle, localisation, compétences, etc.) sont extraits et pré-remplissent la fiche candidat. Si Ollama n’est pas disponible, vous pouvez coller un texte au format NOM: … ROLE: … LOCALISATION: … et utiliser « Pré-remplir quand même ».
  4. Enregistrer : vérifiez la fiche candidat pré-remplie, modifiez si besoin, puis enregistrez. La source sera « VSA » et le lien VSA sera enregistré si vous l’avez indiqué.

Outil Productivité / Matching :

  • Sélection d'entreprises cibles : choisissez une ou plusieurs entreprises pour lesquelles vous cherchez des candidats.
  • Mots-clés automatiques : le système extrait automatiquement les tags techniques de l'entreprise pour générer des mots-clés de recherche. Vous pouvez ajouter/retirer des mots-clés manuellement.
  • Requête LinkedIn : une requête booléenne LinkedIn est générée automatiquement, prête à copier-coller dans LinkedIn Recruiter / Sales Navigator.
  • Matching candidats : score de correspondance entre chaque candidat et les mots-clés sélectionnés.
  • Candidat du jour ⭐ : le meilleur match selon les filtres actuels est mis en avant, avec ouverture directe de son profil LinkedIn.
  • Message d'approche : génération d'un message d'approche personnalisé, prêt à coller.
  • Sauvegarde tags : enregistrez les mots-clés productivité directement dans les tags de l'entreprise pour enrichir les futures recherches.
Ouvrir la page Sourcing →

12) Push & envoi d'emails

Le push désigne l'envoi d'un email/CV à un prospect. Le workflow est optimisé :

  1. Depuis la fiche prospect, cliquez sur 📧 Email.
  2. L'adresse email est copiée automatiquement dans le presse-papier.
  3. Si une catégorie push est associée au prospect, le template Outlook (.msg) correspondant est ouvert automatiquement. Vous n'avez qu'à coller l'email dans le champ "À:".
  4. Le push est loggé automatiquement dans l'historique avec la date, le canal et le template utilisé.

Page Push (historique) :

  • Historique complet de tous les push envoyés, filtrables par recherche et canal.
  • Annuler le dernier push : en cas d'erreur, annulez le dernier envoi.
  • Export Excel : exportez l'historique des push au format .xlsx.
  • Suppression unitaire si besoin.
Ouvrir l'historique Push →

13) Catégories Push

Organisez vos templates d'emails push par compétence / métier :

  • Catégories : chaque catégorie correspond à un dossier dans pushs/ (ex: "C_embarqué", "FPGA", "Electronique"). Le système scanne automatiquement les dossiers existants.
  • Fichiers .msg : chaque catégorie contient des fichiers templates Outlook (.msg, .eml, .oft). Le premier fichier est utilisé comme template par défaut lors de l'envoi.
  • Mots-clés : associez des mots-clés à chaque catégorie pour le matching automatique.
  • Matching candidats : chaque catégorie affiche les Top 3 candidats dont les compétences correspondent le mieux aux mots-clés de la catégorie (score de correspondance).
  • Scan dossiers : le bouton "Scanner" détecte automatiquement les nouveaux dossiers dans pushs/ et crée les catégories correspondantes.
  • Éditeur de templates : créez/modifiez des templates email avec éditeur rich text (gras, italique, liens, variables).

14) Recherche globale

  • Navigation au clavier (↑↓ + Entrée) pour ouvrir directement la fiche.

📄 Onglet Rapport & KPI

Génération de rapports Excel hebdomadaires et checklist KPI :

  • Export suivi d'activité : générez un Excel de suivi hebdomadaire avec :
    • Sélection de la semaine (format ISO : 2026-W10)
    • Option "Enrichir avec l'IA (Ollama)" : normalise les métiers, extrait les besoins depuis les notes, génère les codes notes (B OKS, A OKS, etc.)

    • Date et titre personnalisables
    • Checklist par catégories (Prospection, RDV, Client, Push, Candidats)

    • Modèles rapides : "Tout cocher", "Tout décocher", "Modèle semaine"
    • Export Excel de la checklist
Rappel export — Une alerte vous rappelle chaque jeudi matin si vous n'avez pas encore exporté votre suivi d'activité de la semaine.

16) Rapport hebdomadaire Excel

Générez un rapport Excel de suivi d'activité pour chaque semaine depuis l'onglet "📄 Rapport & KPI" de la page Stats :

  1. Sélectionnez la semaine : utilisez le sélecteur de semaine (format ISO : 2026-W10, ex: "2026-W10" pour la semaine 10). Par défaut, la semaine courante est pré-sélectionnée.
  2. Optionnel : activez l'enrichissement IA : cochez "Enrichir avec l'IA (Ollama)" pour :
    • Normaliser les métiers des candidats (si ambigu ou vide)
    • Extraire les besoins depuis les notes des prospects (clients vus)
    • Générer les codes notes (B OKS, A OKS, etc.) depuis les données EC1
    Note : si Ollama n'est pas disponible, le rapport est généré sans enrichissement (fallback automatique).
  3. Générez le rapport : cliquez sur "📥 Générer le rapport Excel". Le fichier se télécharge automatiquement.

Contenu du rapport Excel

Le fichier Excel contient 17 colonnes avec :

  • Colonnes A-G (Candidats) : Semaine (fusionné), Entretiens (nom), Métier, Exp (années), Dispo, Notes (codes), Commenta (commentaires détaillés)
  • Colonnes H-J (Prospection) : Prospections (RDV pris), Clients vus (RDV effectué), Besoins (extraits depuis notes)
  • Colonnes K-L : RT, Suivi Mission (réservés, vides)
  • Colonnes M-N (Pushs) : Pushs consultant (nom), Nb pushs (nombre)
  • Colonnes O-Q (Objectifs) : Attendus Prosp, Attendus Entretiens, Attendus Pushs (depuis la section Gamification)

Données collectées automatiquement :

  • Candidats EC1/EC2 : tous les candidats vus en entretien (EC1 ou EC2) dans la semaine sélectionnée
  • Prospections : tous les RDV pris avec des prospects (événements rdv_taken ou prospects avec statut='Rendez-vous' et rdvDate dans la semaine)
  • Clients vus : tous les prospects avec statut='Rendez-vous' et lastContact dans la semaine (RDV effectué)
  • Pushs : tous les pushs envoyés dans la semaine, groupés par consultant
  • Objectifs : valeurs depuis la section Gamification (Paramètres → Objectifs & Gamification)
Format Excel — Les cellules "Semaine" sont fusionnées pour toutes les lignes de la même semaine. La colonne G (Commenta) a une bordure épaisse pour servir de séparateur visuel entre les candidats et la prospection.

17) Checklist KPI

Outil de checklist rapide pour générer un fichier Excel de validation, accessible depuis l'onglet "📄 Rapport & KPI" de la page Stats :

  1. Sélectionnez une date : choisissez la date du jour ou de la semaine à valider.
  2. Ajoutez un titre (optionnel) : ex: "Semaine 06", "Validation jour X", etc.
  3. Cochez les items : parcourez la checklist par catégories (Prospection, RDV, Client, Push, Candidats) et cochez les actions réalisées.
  4. Ajoutez des commentaires (optionnel) : pour chaque item, vous pouvez ajouter un commentaire (ex: nom client, contexte, etc.).
  5. Utilisez les modèles rapides :
    • "Tout cocher" : coche tous les items
    • "Tout décocher" : décoche tous les items
    • "Modèle semaine" : coche automatiquement les items typiques d'une semaine (contact, relance, validation, RDV, push, candidat)
  6. Générez l'Excel : cliquez sur "📄 Générer Excel KPI" pour télécharger le fichier .xlsx formaté avec vos validations.
Note — La checklist KPI est maintenant intégrée dans la page Stats (onglet Rapport & KPI) pour centraliser tous vos outils de reporting.

18) Référentiel métiers

Le référentiel contient 251 tags techniques organisés en 15 catégories métier (Électronique, Logiciel embarqué, Informatique industrielle, Mécanique, Systèmes, Automatisme, Robotique, etc.) :

  • Arbre interactif : parcourez les catégories et sous-catégories sous forme d'arborescence dépliable.
  • Recherche : filtrez rapidement pour trouver un tag ou un métier.
  • Statistiques : nombre de tags par catégorie, couverture dans votre base.
  • Tags personnalisés : ajoutez vos propres tags métier en complément du référentiel standard.
  • Utilisation : ces tags sont utilisés partout dans l'app — tags prospects, tags entreprises, compétences candidats, mots-clés de catégories push, et prompts de scrapping IA.

19) Doublons

  • Détection : le système identifie les doublons potentiels par nom, email, ou téléphone.
  • Fusion : pour chaque paire, choisissez la fiche à garder et celle à fusionner. Les données sont consolidées (tags, notes, callNotes, push logs) dans la fiche conservée.
  • Prévisualisation : comparez les deux fiches côte à côte avant de fusionner.

20) Snapshots & restauration

  • Snapshot manuel : créez une sauvegarde instantanée à tout moment.
  • Snapshots automatiques : un snapshot est créé automatiquement au premier lancement de chaque jour.
  • Restauration : restaurez n'importe quelle version. Une sauvegarde "before_restore" est créée automatiquement avant chaque restauration pour sécuriser l'opération.
  • Historique : consultez la liste complète des snapshots avec date et label.
Conseil : faites un snapshot avant une grosse importation ou un nettoyage de données.

21) Paramètres & exports

La page Paramètres regroupe tous les outils de gestion de données :

  • Export JSON : sauvegardez l'intégralité de votre base au format JSON (compatible ré-import).
  • Import JSON : restaurez une base depuis un export JSON précédent.
  • Export Excel : export complet au format .xlsx avec toutes les colonnes.
  • Import tags CSV : importez des tags depuis un fichier CSV pour enrichir vos prospects/entreprises en masse.
  • vCard : export d'un prospect en .vcf depuis la page Prospects (bouton vCard sur une fiche ou export de la sélection), pas depuis Paramètres.
  • Réinitialiser : remise à zéro de la base (avec confirmation).
  • Accès rapide : liens vers Snapshots, Métiers, Doublons, Aide.

22) Mise à jour du serveur

Pour mettre à jour prospup.work après un commit/push sur Git :

  • Sur le PC qui héberge : ouvrez Paramètres (section admin), puis cliquez sur « Mettre à jour et redémarrer ».
  • L'application effectue un git fetch puis git pull sur origin/main, affiche la sortie en direct, puis redémarre le serveur.
  • La page se recharge automatiquement après le redémarrage pour reprendre sur la nouvelle version.
  • Le badge « À propos » (version · branche · commit) permet de vérifier que la mise à jour a bien été appliquée.
