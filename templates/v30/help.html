Démarrage.

Tout ce qu'il faut savoir pour ta première connexion et comprendre comment la v30 s'articule avec la v29.

  1. URL : https://prosphup.work (ou http://localhost:8000 en local).
  2. Connexion : identifiants par défaut admin / admin — à changer au premier passage via Mon compte.
  3. Interface : la v30 est l'interface par défaut depuis la release 30.1. Tu atterris sur /v30/dashboard.
  4. Snapshot auto : une sauvegarde de la base est créée au premier démarrage du jour — restaurable à tout moment depuis Paramètres.
Besoin de l'ancienne interface ? Depuis la sidebar, clique sur « Revenir à l'ancienne interface » (carte dans Paramètres) ou ajoute ?force_v29=1 à n'importe quelle URL. Tes données sont identiques — c'est juste un rendu différent.

{# ─── 2. Dashboard ─────────────────────────────────────── #}
{{ ui.icon('home', 18) }}

Dashboard.

La page d'accueil v30 rassemble en un seul écran ton Action Center, ton pipeline, tes objectifs et les priorités générées par l'IA.

  • Action Center — trois colonnes : Tâches du jour, RDV, Retards. Chaque item est cliquable et ouvre la fiche concernée.
  • Pipeline — barre horizontale par stade (Vivier → Qualifié → Proposé → En entretien → Placé). Les compteurs sont cliquables pour filtrer la liste Prospects.
  • Objectifs & gamification — cibles hebdo (prospects contactés, RDV, pushs). Configuration dans Paramètres → Objectifs.
  • Priorités IA — suggestions générées par Ollama à partir des prospects stagnants (score de pertinence, dernier contact).
  • Fil d'activité — flux temps réel des dernières actions (notes, changements de statut, pushs envoyés).
Astuce : les widgets du dashboard sont déplaçables en drag & drop (desktop) — l'ordre est persisté par utilisateur.

Focus & Calendrier.

Focus te donne une vue chirurgicale des relances à faire. Calendrier te donne la vue mensuelle de tes RDV et échéances.

Focus

  • Trois piles : En retard (rouge), Aujourd'hui (orange), À venir (bleu).
  • Quick actions sur chaque ligne : +1j / +7j / ✓ (marquer fait).
  • Rechargement automatique après chaque action — tu enchaînes sans refresh.

Calendrier

  • Vue mois (lundi → dimanche). Navigation < / > / Aujourd'hui.
  • Événements colorés : 🤝 RDV, 🔄 relance, 📞 EC1/EC2 (entretiens candidat).
  • Popup +N événements quand un jour déborde — clic ouvre directement la fiche correspondante.

Prospects.

Le cœur de l'outil. Trois vues, des filtres qui se souviennent, une bulk bar pour agir sur des dizaines de prospects d'un clic.

Les 3 vues

  • {{ ui.icon('table', 14) }} Table — tableau dense, tri par colonne, toggle colonnes via l'icône engrenage.
  • {{ ui.icon('kanban', 14) }} Kanban — par statut, drag pour changer d'étape.
  • {{ ui.icon('split', 14) }} Split — liste + fiche à droite sans quitter la page.

Filtres & vues enregistrées

  • Pills de vues par défaut : Tous, Mes prospects, À relancer, Chauds.
  • Bouton + pour créer une vue custom à partir des filtres actifs.
  • Persistance localStorage : les filtres sont conservés d'une session à l'autre.

Bulk bar & édition inline

  • Sélectionne plusieurs lignes → la bulk bar flottante propose Tag, Statut, Push, Supprimer.
  • Double-clic sur une cellule → édition inline (nom, fonction, email, téléphone).
  • {{ ui.icon('sparkles', 14) }} Scrapping IA par ligne (icône étoile) — ouvre la modale d'enrichissement.
Clavier : J/K pour naviguer, X pour cocher, E pour éditer en place, ↵ pour ouvrir la fiche.

Fiche prospect.

Tout ce qui concerne un prospect sur une seule page : infos, historique, pushs, enrichissement IA.

  • 4 onglets : Vue d'ensemble, Timeline, Push, IA.
  • Édition inline : clic sur n'importe quel champ (nom, fonction, email, téléphone, notes) pour éditer. Validation par ↵, annulation par Esc.
  • Timeline colorée : points bleu (push), vert (RDV), gris (note), orange (changement de statut).
  • IA drawer : Scrapping (enrichir champs), Avant RDV (fiche prépa PDF), Après RDV (compte-rendu auto → champs mis à jour).
  • Date de prochaine relance : pilote les statuts En retard / Aujourd'hui / À venir dans Focus.
Le bouton Avant RDV IA génère un PDF d'une page (bio LinkedIn + actus + accroche) à imprimer ou partager avant ton rendez-vous.

Entreprises.

Gère les comptes (cibles, clients, prospects chauds) et suis leurs opportunités commerciales.

  • Vues : liste dense ou grille de cartes (toggle haut de page).
  • Filtres : pipeline, avec/sans prospects, tags techniques. KPI en entête (total / en pipeline / prospects liés).
  • Fusion : sélectionne 2 entreprises → Fusionner → wizard de consolidation (sites, tags, prospects).
  • Opportunités : Identification → Qualification → Proposition → Négociation → Gagné / Perdu. Timeline d'événements par opportunité.
  • Export : XLSX multi-colonnes (avec nb de prospects et tags consolidés).
Astuce : centraliser la stack technique dans les tags entreprise (et pas sur chaque prospect) évite les doublons et facilite le sourcing.

IA & Scraping.

L'IA est partout dans l'app : enrichissement de fiches, préparation de RDV, compte-rendus, emails personnalisés, mapping Excel. Moteur local (Ollama) + recherche web optionnelle (Tavily).

Stack

  • Ollama — local, gratuit, http://127.0.0.1:11434. Modèle par défaut : llama3.2. Aucune donnée ne sort de ton PC.
  • Tavily (optionnel, cloud) — recherche web pour enrichir les prompts. Clé configurable dans Paramètres → Configuration IA.
  • Flux web search : Tavily récupère les résultats → Ollama génère en citant ses sources.

Entrées IA dans l'interface

  • Scrapping IA (prospects / entreprises / candidats) — colle une bio, un profil LinkedIn, une page d'entreprise → les champs se remplissent.
  • Avant RDV IA — fiche prépa PDF (bio, actus, accroche).
  • Après RDV IA — compte-rendu en JSON → statut, prochaine relance, notes mis à jour.
  • Email IA / Tel IA — génération bulk depuis la bulk bar.
  • Ajout IA (Quick Add) — 3 étapes (type + contexte → génération → apply) pour créer prospect / entreprise / candidat.
  • Import Excel — Suggérer mapping (header → champ) et Reformater champ (normalisation une valeur par ligne).
Tavily : activer webSearch peut envoyer des infos prospect vers le cloud Tavily. À réserver aux entreprises publiques.
{# ─── 12. Import Excel ────────────────────────────────── #}

Import Excel.

Importe un fichier XLSX et laisse l'IA proposer le mapping des colonnes. Tu valides, tu ajustes, tu importes.

  • Colonnes supportées : Nom, Prénom (combinables), Entreprise, Site, Fonction, Téléphone (fusion multi-colonnes), Email, LinkedIn, Notes, Tags, Pertinence, Statut, Date dernier contact.
  • Auto-mapping via _guessMapping (heuristique sur les en-têtes).
  • Suggestion IA si l'auto-mapping hésite.
  • Reformulation IA par cellule (normaliser fonctions, séparer prénom / nom, nettoyer emails).
  • Fixture de test : tests/fixtures/fichier_prosp_test_user.xlsx.

Paramètres & Admin.

Le hub Paramètres regroupe la configuration IA, les objectifs, les sauvegardes et les actions admin.

Cartes du hub

  • Configuration IA — modèle Ollama, clé Tavily, timeout.
  • Objectifs & gamification — cibles hebdo (prospects contactés, RDV, pushs) affichées sur le dashboard.
  • Sauvegardes & données — snapshots journaliers, export JSON, restauration.
  • Notifications navigateur — push natif pour relances et tâches du jour.
  • Mon compte — changement de mot de passe, profil.
  • À propos — version, commit, release notes.

Admin

  • Mise à jour du serveur (native v30) — git pull origin main + snapshot DB automatique + redémarrage en ~10 s. Logs SSE temps réel.
  • Rollback — revient au commit précédent via .last_commit_hash.
  • Gestion users (/v30/users) — CRUD, assignation de rôles (admin / editor).
  • Journal d'activité (/v30/activity) — filtrable par user et action, lien vers la fiche concernée.
  • Snapshots (/v30/snapshots) — liste, create / restore / delete, progression SSE.
Multi-user : chaque utilisateur a une DB isolée (data/user_<id>/prospects.db). Les admins voient tout ; les editors ne voient que leurs propres enregistrements.
{# ─── 15. Bonnes pratiques ────────────────────────────── #}
{{ ui.icon('bulb', 18) }}

Bonnes pratiques.

Dix habitudes simples pour garder ta base propre et ton pipeline actif.

  1. Renseigne toujours une date de prochaine relance dès la création — c'est ce qui alimente Focus.
  2. Centralise la stack technique dans les tags entreprise (et pas sur chaque prospect).
  3. Lance un Scrapping IA dès qu'une fiche est créée — gain de temps énorme.
  4. Associe tes prospects aux catégories push dès la qualification.
  5. Crée un snapshot avant chaque bulk action destructive.
  6. Sauve tes vues filtrées fréquentes — la prochaine session sera plus rapide.
  7. Utilise ⌘K plutôt que la souris — c'est plus rapide et fatigue moins.
  8. Génère le rapport hebdo le vendredi — auto-injection des KPI, tu n'écris que les notes.
  9. Passe en Mode Prosp 15 min par jour pour avaler le backlog de relances.
  10. Check Focus chaque matin — c'est la vue la plus actionnable de l'app.
